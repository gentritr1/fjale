# Lessons — how each problem was actually solved

A casebook of real problems from FJALË's development, kept so future rounds
(human or agent) can reuse the diagnosis patterns, not just the fixes. Format:
what broke → how the root cause was found → the fix → the transferable lesson.

## Method that produced these

Orchestrator/implementer loop: one agent implements on a feature branch against
a brief with observable acceptance criteria; a reviewer re-executes every claim
(never trusts the report), reproduces the pre-fix failure first, and merges only
on an evidenced verdict. Visual work additionally requires a human eyeball on
screenshots — math never closes a "looks bad" ticket.

## 1. iOS double-tap zoom on the keyboard

- **Problem:** rapid taps on the on-screen keys zoomed the page on iPhones.
- **Diagnosis:** grep showed `touch-action: manipulation` existed only on
  `.board` — the one surface players don't hammer. The viewport meta was
  correct; iOS needs `touch-action` per element, not viewport tricks.
- **Fix:** one line — `touch-action: manipulation` on the shared
  `button, a, select, input` reset. Never `user-scalable=no` (kills pinch-zoom
  accessibility).
- **Lesson:** a property applied "somewhere" is not applied where it matters;
  check the element the user actually touches. Zoom-gesture proof is
  device-only — desktop emulation can verify the property, not the behavior.

## 2. Players stuck on stale versions

- **Problem:** after a deploy, open tabs and installed PWAs kept old code
  indefinitely; "tell users to clear cache" was the wrong instinct.
- **Diagnosis:** a new service worker installs but sits `waiting` until every
  tab closes; nothing surfaced that state.
- **Fix:** update banner — detect the waiting worker, offer "Rifresko", post
  `SKIP_WAITING`, reload on `controllerchange` **guarded by an explicit
  accept flag** (first-install `clients.claim()` also fires that event and
  must never surprise-reload mid-game). Versioned cache purge on activate.
- **Lesson:** users should never need Cmd+Shift+R; the app owns its update
  path. Guard every `controllerchange` reload against the first-install case.

## 3. Update banner broken at phone widths

- **Problem:** the banner rendered one-word-per-line with a button clipped
  off-screen at 375px — shipped that way, verified only at ~800px.
- **Diagnosis:** `position: fixed` + `left: 50%` computes shrink-to-fit width
  against the space right of `left` — only half the viewport. Render-invisible
  at desktop widths, catastrophic at 375px. Missing `flex-wrap` was secondary.
- **Fix:** `inset-inline: 0` + `margin-inline: auto` + `width: fit-content`
  centering, plus `flex-wrap` so actions drop to their own row.
- **Lesson:** every fixed-position overlay gets a forced-visible check at
  320/375px before release (`el.hidden = false` — don't wait for the real
  trigger). Rare-trigger UI escapes routine testing by definition.

## 4. Everyday Albanian words rejected as guesses

- **Problem:** `shokun`, `gjyshja`, `malet`, `detin` — normal inflected forms —
  were "not in the word list." Root cause of most player friction.
- **Diagnosis:** the corpus generator stripped Hunspell flags and kept only
  dictionary roots; the `.aff` file with the declension rules was never
  vendored.
- **Fix:** vendor `sq_AL.aff` from the exact upstream commit whose `.dic`
  matches our pinned checksum; implement deterministic SFX/PFX/cross-product
  expansion (stdlib only); filter to five Albanian tokens; 14,257 → 21,481
  accepted guesses. Guards: probe fixture naming root+flag per word, corpus
  regenerated byte-for-byte in tests, checksums pinned, corpus version
  constant + service-worker bump so caches can't mix releases.
- **Lesson:** call it "declared-form expansion", never "full morphology" — the
  ceiling is what the upstream tagged. Pair vendored data files by upstream
  commit, not by "latest". A determinism test (rebuild == committed) is the
  cheapest possible drift guard.

## 5. Growing the daily pool would rewrite history

- **Problem:** the daily-answer formula depended on pool size; adding word 63
  would silently change every past and future daily answer.
- **Diagnosis:** read the formula: `(dayNumber * step + offset) % poolSize`.
- **Fix:** append-only epoch table — each epoch freezes `{start, poolSize,
  stepBase, offset}`; dates resolve through the epoch active at that date.
  Growth = append a later-dated epoch, provably unable to touch earlier dates
  (a test appends a synthetic 138-word epoch and asserts every launch-era date
  unchanged — and that the new pool genuinely engages, so the proof can't pass
  vacuously).
- **Lesson:** before refactoring scheduling logic, snapshot the complete
  observable behavior (here: 534 date→word rows) and require byte-identical
  output after. "Zero behavior change" is testable; test it.

## 6. Challenge links pointed at array positions

- **Problem:** shared challenge codes encoded the answer's array index, so any
  future catalog edit could silently re-point someone's link to another word.
- **Fix:** immutable `id` per answer (id = historical position, append-only,
  never reused); codes keep the identical wire format but resolve through
  id lookup; the id→word binding is hash-pinned in tests.
- **Lesson:** anything encoded into URLs people share is a public API.
  Decouple it from internal ordering the day you notice, not the day it breaks.

## 7. Post-game helper cards out of rhythm

- **Problem:** the hint card sat visibly tighter under its divider than the
  Besa card on the post-game screen.
- **Diagnosis:** static CSS read as symmetric (shared margins). The rendered
  DOM told the truth: the result panel reorders to the top via `order: -1`,
  but the hint card is still the DOM `:first-child` and kept the reduced
  "leading card" top padding — a state-dependent, structure-level bug.
- **Fix:** one scoped rule: when the result panel is visible, the first
  side-section gets the standard padding.
- **Lesson:** visual-order CSS (`order`, `grid-template-areas`) silently
  divorces `:first-child` from "visually first." Diagnose spacing bugs in the
  rendered DOM with computed values, never by reading the stylesheet.

## 8. Digraph tiles read as second-class letters

- **Problem:** DH/RR/SH tiles looked cramped and lighter than single-letter
  tiles — undermining the product's core claim that a digraph is one letter.
- **Fix:** raise digraph glyphs from 0.8em to 0.95em with slight negative
  tracking; measured the widest digraph at the smallest tile first to prove
  no overflow. Box dimensions untouched.
- **Lesson:** typography changes are cheap to make reversible: one commented
  block, before/after screenshots, and explicit owner sign-off — taste
  decisions belong to humans, measurements to the machine.

## 9. Enter key where no thumb expects it

- **Problem:** submit lived at the keyboard's bottom-left; every phone
  keyboard puts it bottom-right. Players hunted for it.
- **Fix:** row reordered to `… ⌫ ↵`, plus a submit-ready accent: Enter turns
  amber exactly when the fifth letter lands (class toggled in the existing
  render path — no new render triggers).
- **Lesson:** platform conventions beat app-internal logic; and a state-change
  affordance ("you can submit now") is worth more than a static label.

## 10. Environment traps that cost real time

- `localhost:3000` was answered by a *different app* — a stray IPv6 listener
  from another project coexisted with our IPv4 server, and `localhost`
  resolved to the wrong one. Always `curl` the page title and `lsof` the port
  before trusting a preview; bind and browse `127.0.0.1:<uncommon-port>`.
- The default shell Node was v14 while the project needs ≥20; every toolchain
  command silently misbehaves until the nvm path is prepended. Check
  `node -v` before believing any failure.
- A session that crosses midnight changes "today": a daily-puzzle assertion
  that was true in the morning was false by mid-review. Date-dependent
  verification must pin the date it ran at.
